02 — How to call it

Reuse it on a new page

Emit once, then mark anything reveal

style  = cm.motion_css()          # the .reveal primitive
script = cm.motion_js()          # one observer adds .in on entry
# in <head>: cm.motion_noscript()  # stays visible if JS is off

# then, on any element:
<div class="reveal" style="--i:0">…</div>
<div class="reveal" style="--i:1">…</div>   # stagger a group

cm.reveal(html, i=2)               # helper that wraps for you
03 — When to use it

Notes & gotchas

  • Fade + rise, 0.8s, brand easing (cubic-bezier(.2,.75,.2,1)); stagger a group with --i:0,1,2… (×80ms).
  • Opt-in, one observer. Components stay motion-agnostic — a page turns motion on by emitting motion_css() + motion_js() and marking elements .reveal. The brand-page template does this for every section automatically, and the catalog reveals its card groups.
  • Accessible + robust: honors prefers-reduced-motion (shows instantly) and ships a <noscript> fallback so content is never stuck hidden.
  • Reuse across sites: the same three calls on any page. Heavier scroll choreography (hero pin, parallax) stays page-level and opt-in — it's bespoke per layout, not a safe global default.

03 — When to use it

Notes & gotchas

  • Count-adaptive (2–4 steps). Cards get a role, not a fixed index: first = round-left + tooth-right, middle = notch-left + tooth-right, last = notch-left + round-right. So the row always closes — pass 3 and it seals on the right, no editing needed. The blue ramp scales to the count (the 4-step ramp is unchanged).
  • The silhouettes are distinct, not mirrored — each right-hand tooth seats into the next card's left notch; that's the interlock. The art is the original 346×173 Card_Shape drawing (same as the brand pages), stretched with preserveAspectRatio=none, so it matches Patagonia at any card width.
  • Self-contained: process_css() carries every mask + the mobile fallback (single column, rounded-left). Card height is min-height:300px; the grid column count follows --n. Drop it into a .cm-container on any page.
  • Variants are inputs, not copies. Need a 3-step version? Call the same process-row with 3 steps — one handle, one component.
